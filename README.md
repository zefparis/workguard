# WorkGuard — React Native reference app

`workguard-rn` is the **reference architecture** for all HV-GUARD mobile apps
(PlayGuard, SiteGuard, EdGuard, DriveGuard, etc.). Every new Guard clones
this repo and swaps the domain-specific bits (enrollment steps, branding,
API endpoints).

Bare React Native **0.79** + TypeScript. **No Expo.**

## Quick start

```bash
# 1. Install deps (devDependencies required — disable any global omit=dev)
npm install --include=dev --legacy-peer-deps

# 2. Fill in .env
cp .env.example .env
# then edit WORKGUARD_API_KEY and DYNAMSOFT_LICENSE

# 3. iOS
cd ios && pod install && cd ..
npm run ios

# 4. Android
npm run android

# 5. Typecheck (must pass before commit)
npm run typecheck
```


## Enrollment flow (8 steps)

1. **MRZ scan** — Dynamsoft native OCR. Skippable.
2. **Selfie** — VisionCamera front cam, base64 → backend.
3. **Stroop test** — 10 rounds, incongruent word/ink pairs, `performance.now()` reaction time.
4. **Neural reflex** — Go/no-go, VALID/NOISE/GHOST signals, latency + discrimination.
5. **Reaction time** — Simple RT, 5 rounds, anticipation penalty.
6. **Vocal imprint** — Audio Stroop, `fr-FR` STT, phonetic match.
7. **Post-quantum seal** — ML-KEM-768 + ML-DSA-65. Private keys → Keychain/Keystore.
8. **Confirmation** — GuardResult verdict.

Only Step 1 (MRZ) is skippable; the other 7 are mandatory per the reference flow.

## Check-in flow

1. Selfie → face match vs. baseline.
2. ONE randomized cognitive test (Stroop or Reflex).
3. Verdict (AUTHORIZED / DEGRADED / BLOCKED) via `GuardResult`.

## Cloning this template into a new Guard

To spin up `playguard-rn`, `siteguard-rn`, etc.:

1. **Clone** this repo to `<guardname>-rn/`.
2. **Rename** throughout:
   - `package.json` → `name`
   - `app.json` → `name`, `displayName`
   - `android/app/src/main/AndroidManifest.xml` → package
   - `android/app/build.gradle` → `applicationId`
   - `ios/WorkguardRN.xcodeproj`, `ios/WorkguardRN/*` → new project name (Xcode rename)
   - Bundle identifier on both platforms
3. **Adjust enrollment** in `src/screens/Enroll.tsx`:
   - Drop cognitive steps that don't fit the risk model (e.g. SiteGuard keeps Reflex + Reaction, drops Stroop).
   - Add domain-specific steps as new components under `src/components/`.
4. **Update** `src/services/api.ts` endpoints (`/workguard/enroll` → `/<guardname>/enroll`).
5. **Rewrite** `README.md` and `codemagic.yaml` workflow names + bundle IDs.
6. **Re-issue** Dynamsoft license for the new bundle ID.

## Project policies (enforced by convention, not tooling)

- ❌ No Expo. Ever.
- ❌ No mock data. All flows hit the real backend.
- ❌ No direct `fetch` — everything through `src/services/api.ts`.
- ❌ No `Date.now()` for cognitive timing — always `performance.now()`.
- ✅ `npm run typecheck` (tsc --noEmit) must pass before every commit.
- ✅ SignalBus flushes every **1000 ms** — do not change without backend coordination.
- ✅ Post-quantum private keys never leave the device. Stored with
      `ACCESSIBLE.WHEN_UNLOCKED_THIS_DEVICE_ONLY`.

## Deploy (Codemagic)

Both iOS (TestFlight) and Android (APK + AAB) workflows are defined in
`codemagic.yaml`. Codemagic env vars required:

- `API_URL`, `WORKGUARD_API_KEY`, `DYNAMSOFT_LICENSE`, `TENANT_ID`
- `APP_STORE_CONNECT_*` and `CERTIFICATE_PRIVATE_KEY` for iOS signing.

The workflow writes `.env` at build time from Codemagic vars — do **not**
commit a real `.env` to the repo.
